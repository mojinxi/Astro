**前排提醒，装不装ninja都无所谓，那就是个获取cookie用的，并且ninja已经跑路了**



早就想写这篇文章了，但是太懒就耽搁了，今天闲着没事做，而且服务器经常宕机（我一个服务器挂了4个网站，就在我码这篇图文的过程中，服务器就炸了，我还没有备份文件的习惯，心酸啊）我就想着把京东签到给迁移到另一台服务器上，这篇教程也就这么出来了，不对是小记。



# 安装docker



我连接服务器使用的是finalshell。首先要安装docker，这个我已经装了，安装方法在之前的文章里，<a href="https://mojinxi.cn/3.html" class="LinkCard" target="_blank">记一次用Docker部署Aria2-pro</a>



# 部署青龙和ninja



京东签到需要用到青龙面板和ninja，下面我们开始部署



在root文件夹下创建一个目录，目录内创建docker-compose.yml，并且填入以下内容（此处qinglong对应下文容器名称）



```css

version: "3"

services:

  qinglong:

    image: whyour/qinglong:latest

    container_name: qinglong

    restart: unless-stopped

    tty: true

    ports:

      - 5700:5700

      - 5701:5701

    environment:

      - ENABLE_HANGUP=true

      - ENABLE_WEB_PANEL=true

    volumes:

      - ./config:/ql/config

      - ./log:/ql/log

      - ./db:/ql/db

      - ./repo:/ql/repo

      - ./raw:/ql/raw

      - ./scripts:/ql/scripts

      - ./jbot:/ql/jbot

      - ./ninja:/ql/ninja

    labels:

      - com.centurylinklabs.watchtower.enable=false

```

然后保存并退出，命令行里进qinglong文件夹，执行



```css

docker-compose up -d

```



会得到这样一个页面



![](https://dl.img.timecdn.cn/2021/09/21/D9DB8C5C-0325-4CE1-8726-6BE2FB2E0638.png!h.webp)



接下来登录青龙面板，用你服务器的ip加上5700，比如192.168.1.1:5700，在浏览器里登录青龙面板，初始密码都是admin，首次登录会让你修改密码，跟着提示来就行了



![](https://dl3.img.timecdn.cn/2021/09/21/713BB6EF-27EB-47D3-8CA2-6112F8BA49C0.png)



![](https://dl2.img.timecdn.cn/2021/09/21/39BF6E9E-63C1-4BB4-8E9F-EC284A4706A9.png!h.webp)



下面进入容器，也就是qinglong（此处对应上文qinglong，如作修改，此处也要修改）



```css

docker exec -it qinglong bash

```



![](https://dl2.img.timecdn.cn/2021/09/21/9FA35344-ECA1-4344-B2BF-516DD75DCAD9.jpg!h.webp)



再执行以下命令



```css

git clone https://ghproxy.com/https://github.com/shufflewzc/ninja.git /ql/ninja

cd /ql/ninja/backend

pnpm install

pm2 start

```



![](https://dl3.img.timecdn.cn/2021/09/21/2E68A48E-6648-4B27-BBE4-12316BDA07BD.png)



打开青龙configs文件夹的extra.sh文件将以下内容粘贴进去



```css

cd /ql/ninja/backend

pm2 start

```



![](https://dl.img.timecdn.cn/2021/09/21/A2A2FB27-CA2F-490B-B7B0-6FC10E2E0D53.png!h.webp)



因为ninja扫码还是有问题，所以就必须自己更换新的UA，复制下面代码



```css

docker exec -it qinglong bash

cd /ql/ninja/backend

cp .env.example .env

```



然后在文件管理器找到.env文件，按需修改内容，将新的UA放在下面



![](https://dl.img.timecdn.cn/2021/09/21/499A6089-C474-45F4-8547-B17D3ABEAAFA.png!h.webp)



修改完后输入



```css

pm2 start

```

# 结束



### 注意事项



* 重启后务必执行一次`ql extra` 保证 Ninja 配置成功。

* 更新 Ninja 只需要在容器中`ninja/backend` 目录执行`git pull` 然后`pm2 start`

* Qinglong 需要在登录状态（`auth.json` 中有 token）



### Ninja更新方法



打开Finalshell 按顺序复制粘贴代码



```css

docker exec -it qinglong bash

cd /ql/ninja/backend

git pull

pm2 start

```



### 常见问题



Q：为什么我 `git pull` 失败？A：一般是修改过文件，先运行一次 `git checkout .` 再 `git pull`。还是不行就删了重拉。



Q：为什么访问不了？A：一般为端口映射错误/失败，请自行检查配置文件。



Q：为什么访问白屏？A：使用现代的浏览器，而不是古代的。
